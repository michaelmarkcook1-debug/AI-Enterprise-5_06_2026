# UX/UI Guidelines

## Design Principle

The app must feel like an executive-grade decision intelligence platform, not a feature demo.

## Visual Style

Preserve:

- dark navy base
- teal highlights
- gold accents
- premium / sophisticated feel
- clean data cards
- confidence labels
- evidence labels
- provenance badges

Avoid:

- clutter
- too many charts at once
- colourful SaaS dashboard clichés
- hidden critical warnings
- overuse of seed data without warning

## Navigation Principle

Every top-level nav item must answer an executive question.

| Nav | Question |
|---|---|
| Query | What is happening? |
| Understand | What is this? |
| Assess | What should we do? |
| Demonstrate | How do I defend it? |
| Monitor | Is it still valid? |
| Investor Tools | Where should capital go? |

## Page Layout Pattern

Each page should follow:

1. Executive Summary
2. Key Scores / Status
3. Primary Workflow
4. Supporting Evidence
5. Methodology / Provenance
6. Next Action

## Required UI Behaviour

- Major pages must not be endless walls of panels.
- Use collapsible sections for secondary evidence.
- Keep critical warnings visible.
- Every metric should have a label, confidence, and provenance status.
- Every recommendation should have a next action.
- Every page should have a "what this means" summary.

## Copy Style

Use executive language.

Avoid:

- marketing hype
- vendor claims as facts
- overconfident predictions
- vague "AI-powered" language

Prefer:

- "Evidence suggests..."
- "Current estimate..."
- "Reassessment recommended if..."
- "Known limitation..."
- "Board implication..."
