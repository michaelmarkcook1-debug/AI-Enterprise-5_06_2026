# App Audit — UX, Design, Logic (10 June 2026)

Method: app run locally (seed-fallback mode) and walked with Playwright as a
first-time user; all main tabs screenshotted and inspected; code review of the
data flows behind each surface. Investor Tools excluded per scope.

## What works well (keep)
- **Honesty UX is genuinely strong.** The NOT LIVE banner states *why* the data
  isn't live; provenance badges (NOT LIVE: ESTIMATED, seed-confidence) appear at
  panel level. This is the product's differentiation — protect it.
- **The decision-lifecycle nav** (Query → Understand → Assess → Demonstrate →
  Monitor) reads instantly to a first-timer; each tab opens with a plain-English
  question as its kicker.
- **CIO Interpretation panels** on Query give role context next to numbers —
  exactly the "explain the output" pattern the rest of the app should copy.

## Fixed in this pass
1. **Quadrant rendering retired.** `/quadrant` (score × momentum 2×2) now 307s
   to /understand; the "Vendor Leadership Matrix" (Enhance × Innovate 2×2) card
   removed from Understand. Composite multi-category scoring no longer renders
   as a rank or quadrant anywhere reachable.
2. **Analyst Insight** panel added to the top of Query, Understand, Demonstrate
   and Monitor — one paragraph, derived deterministically from the page's own
   data, labelled platform-derived.
3. **Settings page created** (gear icon now points here) with the ingestion
   cost calculator: real verified per-MTok prices, editable workload
   assumptions, per-stage breakdown, staged model strategy. The same estimate
   surfaces beside the Run button in the admin ingestion console.
4. **Four data endpoints stood up** (/api/market-share, /api/uptake,
   /api/reputation, /api/pricing) with stable shapes and provenance fields —
   frontends and external consumers can integrate now; the data source behind
   each repoints to live ingestion after the spend cap lifts without a shape
   change.

## Findings to address (recommendations, not yet implemented)
- **Tab density:** Understand (~10 panels) and Demonstrate (~13 sections) are
  long single scrolls. Recommend a sticky in-page section nav (the pattern
  Query already uses with its side menu) or default-collapsed panels below the
  fold.
- **Route sprawl:** ~14 routes exist outside the nav (atlas, evolution,
  exposure-map, market, methodology, news, reputation, capabilities, briefings,
  vendors, watchlists, dashboard, results, classic query). First-time users
  can't discover them; maintainers must maintain them. Prune or index.
- **Console errors on Understand (35×):** external resource fetches fail
  offline (ERR_NAME_NOT_RESOLVED — likely vendor logos). Needs a graceful
  local fallback so offline/firewalled environments don't degrade.
- **Hover explanations:** scores lack tooltips explaining derivation. The
  Analyst Insight panel partially covers this at page level; per-cell tooltips
  on the strategic table and drift table are the next increment.
- **Trend visuals:** momentum and drift data exist and could power sparklines
  today. TRUE 12-month rolling growth needs ranking-snapshot history that is
  not yet accumulated — do not fake it; start persisting snapshots now so the
  feature becomes honest in ~a quarter.
