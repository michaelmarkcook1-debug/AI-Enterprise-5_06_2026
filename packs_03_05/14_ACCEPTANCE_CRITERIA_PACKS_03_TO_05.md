# Acceptance Criteria for Packs 03–05

## Pack 03 Pass Criteria

- Source registry exists.
- Evidence object model exists.
- Evidence has grade, confidence, freshness, licence and review status.
- Claim publishability status exists.
- Source conflict handling exists.
- Scores have methodology definitions.
- Seed/estimated/live distinctions are visible.
- Admin can inspect evidence quality.

## Pack 04 Pass Criteria

- Export centre exists.
- Demonstrate can produce a board-pack-style export.
- Export includes executive summary.
- Export includes recommendation.
- Export includes risk register.
- Export includes assumptions.
- Export includes evidence appendix.
- Export includes methodology appendix.
- Export labels seed/estimated data.

## Pack 05 Pass Criteria

- Product editions are defined.
- Role model exists.
- Workspace concept exists.
- Saved assessment concept exists.
- Saved board pack concept exists.
- Audit event concept exists.
- Enterprise readiness checklist exists.
- Legal/data-use disclaimers exist or are planned clearly.

## Final Product Readiness Gate

The app is not externally publishable unless:

1. Public claims are evidence-backed.
2. Source usage is legally safe.
3. Estimated data is labelled.
4. Methodology is visible.
5. Exports include evidence and caveats.
6. User roles and data controls are defined.
7. Data refresh/staleness is visible.
8. The product does not overclaim certainty.
